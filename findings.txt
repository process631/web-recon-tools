2. Vulnerability Analysis
--------------------------------------------------------------------------------

SQL injection (blind)
  The /adminlogin.do endpoint is vulnerable to Boolean-based SQL injection via
  the username parameter.

  CAPTCHA (yzm): Bypassing the CAPTCHA may be possible by omitting the yzm
  parameter from the request (document evidence per lab requirements).

  Initial automation returned "aaaaaaaa" strings, suggesting false positives
  where the WAF or backend returns HTTP 500 for complex SQL syntax.

Local file inclusion (LFI)
  A potential LFI was identified in the file= parameter of login.jsp.

Sensitive directory exposure
  ffuf and dirb scans indicated paths such as /admin, /FCKeditor, /images,
  /Scripts, and /upload_file (often as redirects; verify with follow-up).

  Access to /.git/config and /.svn/entries returned HTTP 403 Forbidden
  (X-Panyun-Error-Step: 5). Interpretation: blocking/denial at the edge;
  correlate with full response headers in evidence appendix.

Insecure session management
  The JSESSIONID cookie is missing the HttpOnly flag (per nmap http-cookie-flags),
  which can increase impact if XSS or script-access scenarios exist.
